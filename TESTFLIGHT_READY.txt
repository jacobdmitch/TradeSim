🐛 COMMON ISSUES & FIXES

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ISSUE: Can't archive (no device option)
FIX:   Select "Any iOS Device (arm64)" in scheme selector
       NOT a simulator!

ISSUE: Signing failed
FIX:   Xcode → Settings → Accounts → Download Manual Profiles
       Enable "Automatically manage signing" in target settings

ISSUE: Missing app icon warning
FIX:   Run ./generate_icons.sh
       Import all sizes to Assets.xcassets
       Ensure 1024×1024 App Store slot is filled

ISSUE: Upload stuck or failed
FIX:   Check internet connection
       Verify Apple system status
       Try again after a few minutes
       Archive again if needed

ISSUE: Build stuck "Processing" > 1 hour
FIX:   Check email for error messages
       Re-upload if no email after 2 hours
       Verify all files included in archive

ISSUE: Beta review rejected
FIX:   Read rejection email carefully
       Common issues:
       - Missing privacy policy
       - Unclear app purpose
       - Missing permissions explanations
       Address issues and resubmit

💡 PRO TIPS

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. TEST FIRST: Always test on physical device before uploading

2. VERSION CONTROL: Use Git to track changes
   git tag v1.0.0 before each major release

3. INCREMENTAL BUILDS: Increment build number for each upload
   Version 1.0.0 can have builds 1, 2, 3, etc.

4. INTERNAL FIRST: Start with internal testing (faster)
   Then move to external when ready

5. PUBLIC LINK: Easiest way to share with many testers
   No need to collect email addresses

6. FEEDBACK: Encourage testers to use TestFlight feedback
   Shows up directly in App Store Connect

7. CRASHES: Monitor crash reports and fix ASAP
   High crash rate can affect App Store approval

8. UPDATES: Keep testers engaged with regular updates
   Even small improvements show progress
